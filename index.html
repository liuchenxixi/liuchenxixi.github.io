ctf升级之路——SQL注入方法总结

It is the first time for me to display my blog on-line by using the markdown’s language. My blog content may have a few bugs and flaws, and it may not look pretty since it’s my first made. If there is a shortage, please do not hesitate to give advice, welcome everyone to correct it.

什么是SQL注入

SQL注入是一种将sql代码添加到参数中，传递到SQL服务器解析并执行的一种攻击手法。它是发生在Web程序中数据库层的安全漏洞，是网站存在最多也是最简单的漏洞。

其产生原因是程序未对用户输入数据的合法性进行判断和处理，导致攻击者可以在Web应用程序这事先定义好的SQL语句中添加额外的SQL语句， 在管理员不知情的情况下实现非法操作，以此来实现欺骗数据库服务器执行非授权的任意查询，从而进一步获取到数据信息。

sql语句

SELECT * FROM admin WHERE user = “test” AND pass = “123456”;

意为从admin表中查找user为test并且pass为123465的记录，并将满足要求的记录输出

  1. 输入密码“ OR ”1“=”1，用户名为test，则WHERE后的表达式一定返回true，于是mysql会将每条记录都输出，而网站误认为此用户名正确，然后让我以test的身份登录
  2. 若网站还设有管理权限，还可尝试以**” OR “1”=”1” AND writable = TRUE AND “”=”**为密码

Tool

sqlmap:支持MySQL, Oracle,PostgreSQL, Microsoft SQL Server, Microsoft Access, IBM DB2, SQLite, Firebird,Sybase和SAP MaxDB等数据库的各种安全漏洞检测。

sqlmap支持五种不同的注入模式：

  1. 基于布尔的盲注（可以根据返回界面判断条件真假的注入）
  2. 基于时间的盲注（不可根据页面返回内容判断任何信息，用条件语句查看时间延迟语句是否执行，也就是页面返回时间是否增加来判断）
  3. 基于报错注入（页面会返回错误信息，或者把注入的语句的结果直接返回在界面中）
  4. 联合查询注入 （可以使用union的情况下的注入）
  5. 堆查询注入 （可以同时你执行多条语句的执行时的注入）

使用参数

1 普通选项-h,–help 显示基本帮助信息并退出-hh 高级帮助信息并退出

:-version 显示程序版本信息并退出2 确定目标

-d DIRECT 直接连接数据库的连接字符串

-u URL,–url=ULR 目标URL(网址？)

-I LOGFILE 从Burp

Sqlmap直连数据库

  1. 服务型数据库（前提知道数据库用户名和密码）：*DBMS://USER:PASSWORD@DBMS_IP:DBMS_PORT/DATABASE_NAME(MySQL,Oracle,Microsoft SQL Server，PostgreSQL,etc.)**eg:**python sqlmap.py -d “mysql://admin@192.168.21.17:3306/testdb” -f –banner –dbs –users

  2. 文件型数据库（前提知道数据库绝对路径）：

    DBMS://DATABASE_FILEPATH(SQLite,Microsoft Access,Firebird,etc.)

Sqlmap URL探测

  1. **单一URL探测，参数使用-u 或–url URL格式：http(s)://targeturl[:port]/[……] (:port端口可能不是默认的80端口

Sqlmap文件读取目标

*Sqlmap支持从不同类型的文件中读取目标进行Sqlmap注入探测
  1. -l从Burpsuite proxy或WebScarab proxy中读取Http请求日志文件
  2. -x从sitemap.xml站点地图文件中读取目标探测
  3. -m从多行文本格式文件读取多个目标，对多个目标进行探测
  4. -r从文本文件内中读取Http请求作为Sql注入探测的目标
  5. -c从配置文件sqlmap.conf中读取目标探测
